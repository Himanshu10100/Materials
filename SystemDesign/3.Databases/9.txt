# üîµ **1. What is SQL? (Relational Databases)**

SQL databases store data in **tables with rows and columns**.

Examples:

* MySQL
* PostgreSQL
* Oracle
* Microsoft SQL Server
* MariaDB

### **SQL = Structured + Strong consistency (ACID).**

---

## üü¢ **SQL Data Model**

```
Users Table:
+----+----------+------------+
| id | name     | email      |
+----+----------+------------+
| 1  | Himanshu | h@gmail.com|
```

### Features:

‚úî Schema (fixed structure)
‚úî Joins
‚úî Transactions
‚úî ACID guarantees

---

# üîµ **2. What is NoSQL? (Non-relational Databases)**

NoSQL databases store data in **flexible formats** such as:

* Key-value
* Document
* Wide column
* Graph

Examples:

* MongoDB
* Cassandra
* DynamoDB
* Redis
* Neo4j

### **NoSQL = Flexible + Highly scalable (BASE).**

---

# üü£ **NoSQL Data Models**

## **1. Document Store (MongoDB)**

Stores JSON-like documents:

```json
{
  "id": 1,
  "name": "Himanshu",
  "skills": ["Node", "React"]
}
```

## **2. Key-Value Store (Redis, DynamoDB)**

```
"user:1" ‚Üí { id:1, name:"Himanshu" }
```

## **3. Wide Column Store (Cassandra)**

* Like Google BigTable
* Used for large-scale analytics, logs

## **4. Graph Store (Neo4j)**

* Social networks
* Recommendation engines

---

# #Ô∏è‚É£ **3. Core Differences: SQL vs NoSQL**

| Feature        | SQL                       | NoSQL                       |
| -------------- | ------------------------- | --------------------------- |
| Data Model     | Tables                    | Document/Key-value/Graph    |
| Schema         | Fixed                     | Dynamic/Flexible            |
| Consistency    | ACID                      | BASE (eventual consistency) |
| Scaling        | Vertical (bigger machine) | Horizontal (more machines)  |
| Joins          | Strong support            | Weak/no support             |
| Query Language | SQL                       | Depends on DB               |
| Best For       | Complex queries           | Fast, large-scale apps      |
| Speed          | Slower for huge scale     | Very fast at scale          |

# #Ô∏è‚É£ **11. Interview-Level Comparisons**

### **Q1: SQL vs NoSQL?**

> SQL is relational, ACID-compliant, schema-based, and vertically scalable.
> NoSQL is non-relational, flexible, BASE-based, and horizontally scalable.

### **Q2: Can NoSQL replace SQL?**

> No. They solve different problems.
> SQL = transactions & consistency
> NoSQL = speed & scalability

### **Q3: Why NoSQL is faster?**

> It avoids joins, enforces no strict schema, and uses distributed architecture.

### **Q4: Does NoSQL support transactions?**

> Yes, modern DBs (MongoDB, DynamoDB) support *limited transactions*, but not as strong as SQL.
