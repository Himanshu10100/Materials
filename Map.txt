Below is a unified roadmap based on how big tech & top product companies typically interview fresh/early-experience engineers.

## **1. Data Structures & Algorithms (DSA) ‚Äì 60‚Äì70% weight**

Topics to cover in order of priority:

### **Core**

* Arrays & Strings
* Hashing
* Sliding Window
* Two Pointers
* Linked List
* Stack/Queue
* Binary Search
* Recursion + Backtracking
* Trees + BST

### **Difficulty Distribution**

* **Easy:** 40%
* **Medium:** 50%
* **Hard:** 10% (optional unless targeting FAANG)

| Company                     | Difficulty Trend | What to Focus                           |
| --------------------------- | ---------------- | --------------------------------------- |

## **2. Low-Level Design (LLD) ‚Äì mandatory for product companies**

Topics:

* OOP Principles (solid understanding)
* Design Patterns
* Designing:

  * Parking Lot
  * Splitwise
  * BookMyShow
  * Snake & Ladder
  * Elevator System
  * Rate Limiter

---

## **3. High-Level System Design (HLD) ‚Äì light/basic for 1‚Äì2 YOE**

Understand:

* Load balancers
* Caching (Redis)
* Message queues
* Database sharding
* Basic designs like:

  * URL Shortener
  * Instagram Feed
  * Chat application

---

## **4. Computer Science Core**

Companies like Google/Microsoft test this more:

* OS concepts
* Networking
* DBMS + SQL
* Concurrency basics

---

## **5. Engineering / Project Round**

Prepare:

* Real-world project explanation
* System components
* API design
* Scalability decisions
* Performance improvements
